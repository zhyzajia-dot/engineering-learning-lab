## 1. 概述

本算法基于毫米波雷达传感器数据，通过实时信号处理、多维度评分函数和非对称状态机，实现睡眠分期分析。算法同时运行在 Python（离线分析/仿真）和 C++（ESP32 嵌入式端），两者逻辑完全一致。

**核心设计原则：**

- **非对称状态机**：入睡困难、觉醒容易（人体睡眠自然特性）
- **双时间尺度体动 EMA**：慢速 EMA（~5分钟）用于入睡/深睡判断，快速 EMA（~3秒）用于觉醒检测
- **相对稳定性判定**：不依赖绝对阈值，依据信号相对变化评估睡眠深度

## 2. 数据采集与输入字段

### 2.1 雷达传感器原始字段

| 字段名 | 类型 | 取值范围 | 说明 |
|--------|------|----------|------|
| `heartRate` | float | 40~120 bpm | 雷达提取的心率 |
| `breathingRate` | float | 8~24 bpm | 雷达提取的呼吸率 |
| `bodyMovement` | float | 0~100 | 体动幅度（雷达 0x03 帧） |
| `sleepState` | int | -1 / 0~3 | 雷达原生睡眠分期：0=清醒, 1=浅睡, 2=深睡, 3=快速眼动, -1=无数据 |
| `bedStatus` | int | 0 / 1 | 床位占用状态：0=离床, 1=在床上 |
| `personDetected` | int | 0 / 1 | 人员检测标志 |
| `humanDistance` | float | cm | 人体距离（雷达测距） |
| `humanActivity` | int | 0=无人 / 1=静止 / 2=活跃 | 人员活动等级 |
| `heartbeatWaveform` | float | 原始波形 | 心跳波形幅值（用于 HRV 估算） |
| `breathingWaveform` | float | 原始波形 | 呼吸波形幅值（用于体动回退估计） |
| `abnormalState` | int | 0 / 1 | 异常状态标志 |

### 2.2 数据采样策略

- **采样频率**：1 Hz（每秒 1 个采样点）
- **时间对齐**：各字段稀疏时间序列通过**前向填充（Forward Fill）** 对齐到均匀 1Hz 时间线
- **ESP32 端**：直接使用雷达串口原始数据帧，`data_processor.cpp` 负责平滑处理

### 2.3 体动数据优先级

```
bodyMovement > humanActivity > 波形方差回退估计
```

1. 优先使用 `bodyMovement`（雷达 0x03 帧，0~100）
2. 若无 `bodyMovement`，将 `humanActivity` 映射为体动值（0→0, 1→2, 2→30）
3. 若以上均无，从 `heartbeatWaveform`/`breathingWaveform` 的 IQR 方差推导体动

## 3. 数据预处理层

### 3.1 生理信号平滑（PhysioProcessor）

模拟嵌入式 `data_processor.cpp` 的平滑逻辑，使用 EMA 滤波平滑心率/呼吸率原始值。

```
EMA_HR_ALPHA = 0.3
EMA_RR_ALPHA = 0.3

HR_smoothed = 0.3 × HR_raw + 0.7 × HR_smoothed_prev
RR_smoothed = 0.3 × RR_raw + 0.7 × RR_smoothed_prev
```

- 维护 60 秒历史窗口用于后续 HRV / RR 变异性计算
- 有效性判定：平滑后值 > 0 视为 valid

### 3.2 心率变异性估算（HRV）

从心跳波形幅值计算 RMSSD（相邻差分均方根）：

```
RMSSD = sqrt( Σ(diff_i²) / N )

其中 diff_i = |waveform_i - waveform_{i-1}|
```

- 使用 60 秒窗口内的波形幅值序列
- RMSSD > 0 视为 valid

### 3.3 呼吸变异性与规律性

从呼吸率历史窗口（≥5 个点）计算：

```
RR_mean  = Σ(RR_i) / N
RR_var   = sqrt( Σ((RR_i - RR_mean)²) / N )
RR_reg   = 1 - min(RR_var / 5.0, 1.0)    // 规律性：越稳定越接近 1
```

### 3.4 体动回退估计（波形 IQR）

当 `bodyMovement` 和 `humanActivity` 都不可用时，从波形 IQR 推导：

```
Window    = 30 秒
hbw_iqr   = Q3(hb_waveform) - Q1(hb_waveform)
brw_iqr   = Q3(br_waveform) - Q1(br_waveform)
combined  = max(hbw_iqr, brw_iqr)
raw       = min(combined / 3.0, 100.0)

Movement_EMA = 0.05 × raw + 0.95 × Movement_EMA_prev
```

IQR（四分位距）替代方差，抗尖峰干扰，比单纯方差更稳健。

## 4. 归一化层

将所有生理参数映射到 [0, 1] 或 [-1, 1] 区间，确保不同量纲的信号可公平参与综合评分。

### 4.1 心率归一化

```
HR_norm = constrain((HR - baselineHR) / 20.0, -1.0, 1.0)
```

- 解译：HR 等于基线时 → 0；每偏离基线 20 bpm → ±1

### 4.2 呼吸率归一化

```
RR_norm = constrain((RR - baselineRR) / 4.0, -1.0, 1.0)
```

- 解译：RR 等于基线时 → 0；每偏离基线 4 bpm → ±1

### 4.3 心率变异性归一化

```
HRV_norm = constrain(HRV / 50.0, 0.0, 1.0)
```

- 解译：HRV 越高，归一化值越大（越接近深睡特征）

### 4.4 体动归一化（非线性开平方映射）

```
Movement_norm = constrain((Movement / 100.0)^0.5, 0.0, 1.0)
```

**核心设计**：非线性归一化——压缩高值、放大小值。

| Movement 原始值 | 实际含义 | 线性归一化 | 开平方归一化 | 效果 |
|----------------|---------|-----------|-------------|------|
| 0~3 | 非常静止 | 0~0.03 | 0~0.17 | 小动作被放大 |
| 3~10 | 微小动作 | 0.03~0.10 | 0.17~0.32 | 微动可感知 |
| 10~20 | 正常翻身 | 0.10~0.20 | 0.32~0.45 | 翻身明显贡献 |
| 20~40 | 明显活动 | 0.20~0.40 | 0.45~0.63 | 接近觉醒 |
| 40~60 | 接近觉醒 | 0.40~0.60 | 0.63~0.77 | 大动作 |
| 60~80 | 清醒大动作 | 0.60~0.80 | 0.77~0.89 | 保留大动作优势 |

## 5. 评分函数层

五个评分函数输出 0~1 之间的分值，分别评价当前秒属于某睡眠状态的可能性。

### 5.1 基础工具函数

**Sigmoid 激活：**

```
Sigmoid(x) = 1 / (1 + e^(-x))
```

将任意实数映射到 (0, 1)，实现平滑的非线性评分。

**EMA 平滑（所有评分）：**

```
Score = 0.2 × RawScore + 0.8 × PrevScore
```

时间常数约 5 秒，防止评分剧烈抖动。

---

### 5.2 入睡倾向评分（Sleepiness Score）

**用途**：判断用户是否即将入睡/已经入睡。用于 IN_BED → LIGHT_SLEEP 和 AWAKE → LIGHT_SLEEP 的切换。

**运算公式：**

```
hr_factor = if HR_valid: (1 - HR_norm) × 0.5  else: 0.5
hrv_n     = if HRV_valid: HRV_norm             else: 0.0
rr_stable = if RR_valid: 1 - min(|RR - baselineRR| / 5.0, 1.0) else: 0.0
move_n    = if Movement_valid: Movement_norm(Movement_sleep_EMA) else: 0.0

x = 3.0 × (hr_factor - 0.5)
  + 2.5 × (hrv_n - 0.5)
  + 2.0 × (rr_stable - 0.5)
  + 3.0 × (0.5 - move_n)         // 慢速 EMA：长期低活动加分

Sleepiness = Sigmoid(x)
```

**关键设计：**
- 使用**慢速体动 EMA**（α=0.01，~5分钟平均）：仅长期静止才判高困倦
- 权重分配：体动(3.0) = HR(3.0) > HRV(2.5) > RR稳定性(2.0)

---

### 5.3 深睡评分（Deep Sleep Score）

**用途**：判断是否处于深度睡眠。用于 LIGHT_SLEEP → DEEP_SLEEP 切换。

**运算公式：**

```
// 硬体动上限：大动作直接否决
if Movement > 25: return 0.0

hr_factor = if HR_valid: (1 - HR_norm) × 0.5  else: 0.5
hrv_n     = if HRV_valid: HRV_norm             else: 0.5
move_n    = if Movement_valid: Movement_norm(Movement_sleep_EMA) else: 0.0

x = 5.0 × (hr_factor - 0.5)    // HR 稳定性权重最高
  + 1.5 × (0.5 - hrv_n)        // HRV 权重降低（信号可能不可靠）
  + 3.0 × (0.5 - move_n)       // 使用慢速 EMA：长期低体动

DeepScore = Sigmoid(x)
```

**相对稳定性判定**：不再要求极低绝对体动/HR，而是看 HR 低于近期平均 + 体动稳定低水平。

**进入深睡前置条件：**
- 浅睡需持续稳定 ≥ 20 分钟（`LIGHT_SLEEP_STABILITY_MIN_SEC = 1200`）
- 深睡评分 > 0.4 稳定 ≥ 180 秒（`DEEP_SLEEP_CONFIRM_SEC`）

---

### 5.4 浅睡评分（Light Sleep Score）

**用途**：判断是否处于浅睡状态。作为"中间稳定态"，各种特征接近中值。

**运算公式：**

```
hr_factor = if HR_valid: (1 - HR_norm) × 0.5                else: 0.0
hr_mid    = 1 - |hr_factor - 0.5| × 2             // 接近中线得分高

hrv_mid   = if HRV_valid: 1 - |HRV_norm - 0.5| × 2         else: 0.0

move_mid  = if Movement_valid and 0.1 ≤ move_n ≤ 0.4:
              1 - |move_n - 0.25| × 4             // 中等体动得分高
            else: 0.0

rr_stable = if RR_valid: 1 - min(|RR - baselineRR| / 5.0, 1.0) else: 0.0

Light = 0.3 × hr_mid + 0.3 × hrv_mid + 0.2 × move_mid + 0.2 × rr_stable
```

**设计特点**：追求各项指标的"中间值"，HR 适中、HRV 适中、体动适中、RR 稳定。

---

### 5.5 觉醒评分（Awake Score）

**用途**：检测觉醒/清醒状态。用于任意睡眠状态 → AWAKE 的切换。

**运算公式：**

```
move_n  = if Movement_valid: Movement_norm(Movement_awake_EMA) else: 0.0
hr_awake = if HR_valid: (HR_norm + 1.0) × 0.5                    else: 0.5
rr_v     = if RR_valid: min(RR_var / 5.0, 1.0)                    else: 0.0

x = 5.0 × (move_n - 0.20)     // 快速体动为主（降低偏置，更激进）
  + 1.5 × (hr_awake - 0.5)
  + 1.0 × (rr_v - 0.3)

AwakeScore = Sigmoid(x)
```

**关键设计：**
- 使用**快速体动 EMA**（α=0.3，~3秒响应）：对瞬时体动突峰极度敏感
- 快速触发路径：`Movement > 45` → 即使只有 1 帧，立即切换到 AWAKE（不走 EMA 评分通道）
- 慢速触发路径：`AwakeScore` 持续为最高，确认 30 秒后切换

---

### 5.6 快速眼动睡眠评分（REM Score）

**用途**：判断是否处于快速眼动睡眠。特征：HR 较高、HRV 较低、RR 不稳定但体动低。

**运算公式：**

```
// 大动作直接否决
if Movement > 25: return 0.0

hr_high      = if HR_valid:     max(HR_norm, 0.0)                          else: 0.0
hrv_low      = if HRV_valid:    1.0 - HRV_norm                              else: 0.0
rr_unstable  = if RR_valid:     min(|RR - baselineRR| / 3.0, 1.0)          else: 0.0
rr_change    = if RR_valid:     min(|RR - lastRR| / 3.0, 1.0)              else: 0.0
move_low     = if Movement_valid: 1.0 - Movement_norm(Movement)             else: 0.0

x = 2.5 × (hr_high - 0.3)
  + 2.0 × (hrv_low - 0.3)
  + 1.5 × (rr_unstable - 0.3)
  + 2.0 × (rr_change - 0.3)    // RR 逐秒变化大是 REM 特征
  + 2.0 × (move_low - 0.5)

REMScore = Sigmoid(x)
```

**REM 特征模式**：
- 心率相对较高（接近清醒水平）
- HRV 较低（自主神经活跃）
- 呼吸率不稳定（RR 方差大、逐秒变化大）
- 体动极低（肌肉弛缓）—— 但注意区分深睡（体动也低但 HR 低、RR 稳）

## 6. 状态机层

### 6.1 状态枚举（9 种状态）

```
NO_PERSON(0) → IN_BED(1) → AWAKE(2) → LIGHT_SLEEP(3) → DEEP_SLEEP(4)
                                                          ↕
                                                    REM_SLEEP(5)
                                                          ↓
                                              OUT_OF_BED(6) → GETTING_UP(7)
                                                          ↓
                                                   SESSION_END(8)
```

### 6.2 完整状态切换规则

#### NO_PERSON → IN_BED / OUT_OF_BED

```
if 检测到人体 (HR 或 RR 有效，或 presence==1):
    if distance > 80 cm:
        → OUT_OF_BED
    else:
        → IN_BED
```

#### IN_BED 状态流转

```
输入：bedStatus, 困倦度, 慢速体动 EMA, 快速体动 EMA

1. bedStatus==0 → 无条件 → OUT_OF_BED

2. 无人检测 > 600s 且无 HR → SESSION_END
3. 无人检测 > 30s → OUT_OF_BED

4. 入睡条件（同时满足 300s）:
   - Sleepiness > 0.45
   - Movement_sleep_EMA < 8
   → 进入 LIGHT_SLEEP

5. 快速觉醒:
   - Movement_awake_EMA > 25
   → 进入 AWAKE
```

#### AWAKE 状态流转

```
1. bedStatus==0 → OUT_OF_BED

2. 无人检测 > 600s → SESSION_END

3. 起床检测（曾入睡 && Movement ≥ 30 持续 300s）:
   → GETTING_UP

4. 重新入睡（同时满足 300s）:
   - Sleepiness > 0.45
   - Movement_sleep_EMA < 8
   → LIGHT_SLEEP
   若清醒时长 ≥ 120s 且曾入睡 → wakeCount++
```

#### LIGHT_SLEEP 状态流转

```
1. bedStatus==0 → OUT_OF_BED

2. 快速觉醒：Movement > 45 → AWAKE

3. HR 持续升高（高于睡眠基线 8bpm 或 12%，累计 120s）→ AWAKE

4. AwakeScore 最高（确认 30s） → AWAKE

5. DeepScore 最高 && > 0.4 && 浅睡稳定 ≥ 20min:
   体动 < 25 稳定 ≥ 60s && 确认 180s → DEEP_SLEEP

6. REMScore 最高 && > 0.45（确认 60s）→ REM_SLEEP
```

#### DEEP_SLEEP 状态流转

```
1. bedStatus==0 → OUT_OF_BED

2. 快速觉醒：Movement > 45 → AWAKE

3. HR 持续升高 → AWAKE

4. 体动 > 25（确认 30s）→ LIGHT_SLEEP

5. 体动 > 25（确认 15s）→ AWAKE

6. REMScore > LightScore && REMScore > 0.3 && DeepScore < 0.3:
   确认 60s → REM_SLEEP
```

#### REM_SLEEP 状态流转

```
1. bedStatus==0 → OUT_OF_BED

2. 快速觉醒：Movement > 45 → AWAKE

3. HR 持续升高 → AWAKE

4. AwakeScore 最高（确认 30s）→ AWAKE

5. LightScore 最高 && REMScore < 0.3 → LIGHT_SLEEP

6. DeepScore 最高 && > 0.4 && 稳定 ≥ 60s（确认 180s）→ DEEP_SLEEP
```

#### OUT_OF_BED / GETTING_UP / SESSION_END

```
OUT_OF_BED:
  有人 → IN_BED
  无人 > 600s → SESSION_END

GETTING_UP:
  无人 > 600s → SESSION_END
  体动 < 30 持续 60s → AWAKE（重新躺下）

SESSION_END:
  有人 → IN_BED（新会话）
```

## 7. 统计与输出层

### 7.1 基础统计

**累计方式**：每秒根据当前状态累加对应的时长计数器。

| 统计项 | 数据来源 | 说明 |
|--------|---------|------|
| `totalSleepTime` | LIGHT + DEEP + REM | 总睡眠时长 |
| `deepSleepTime` | DEEP_SLEEP 状态累计 | 深睡时长 |
| `lightSleepTime` | LIGHT_SLEEP 状态累计 | 浅睡时长 |
| `remSleepTime` | REM_SLEEP 状态累计 | 快速眼动时长 |
| `awakeTime` | AWAKE + OUT_OF_BED（入睡后） | 清醒时长 |
| `sleepStartTime` | 首次进入浅睡的时间戳 | 入睡时间 |
| `sleepEndTime` | 最后一次处于睡眠状态的时间戳 | 睡眠结束时间 |
| `sleepLatency` | sleepStartTime - sessionStartTime | 入睡耗时 |
| `wakeCount` | 入睡后清醒 ≥ 120s 次数 | 醒来次数 |

### 7.2 睡眠周期计数

```
周期开始 = 首次进入 DEEP_SLEEP 的时间

周期切换 = 完成 DEEP 阶段 + 完成 REM 阶段后进入 AWAKE/OUT_OF_BED

每当 "清醒/离床" 出现在 "已完成深睡或已完成 REM" 之后 → cycleCount++
```

### 7.3 睡眠评分模型（满分 100）

| 维度 | 满分 | 评分逻辑 |
|------|------|----------|
| **时长评分** | 18 | 7~9h → 18；6~7h 或 9~10h → 13；<6h 或 >10h → 5 |
| **深睡评分** | 14 | 深睡比例>20% → 14；>15% → 11；>10% → 7；<10% → 3 |
| **连续性评分** | 11 | 醒来≤1次 → 11；≤3次 → 7；≤5次 → 4；>5次 → 2 |
| **生理质量** | 7 | 当前固定值 7（预留校准接口） |
| **入睡速度** | 8 | <20min → 8；<30min → 6；<45min → 3；≥45min → 1 |
| **睡眠效率** | 14 | >90% → 14；>80% → 10；>70% → 6；≤70% → 3 |
| **周期评分** | 28 | ≥4周期→20；≥3周期→15；≥2周期→10；≥1周期→6 + REM占比 20~25% → +8；≥15% → +5；<15% → +2 |

**总分** = Σ各维度 / 100 × 100（clamp 0~100）

### 7.4 输出产物

| 输出 | 格式 | 说明 |
|------|------|------|
| 终端日志 | Console | 仅状态切换时打印：时间/HR/RR/体动/算法状态/雷达状态/评分 |
| CSV 结果 | `*_sleep_result.csv` | 逐秒分期结果，含评分和雷达对比 |
| 综合报告图 | `*_report.png` | 饼图 + 统计卡片 + 柱状图 + 睡眠分期图 + 热力图 |

## 8. 参数总表

### 8.1 EMA 平滑参数

| 参数 | 值 | 用途 |
|------|----|------|
| `EMA_ALPHA` | 0.2 | 评分函数 EMA 平滑（~5秒） |
| `EMA_SLEEP_ALPHA` | 0.01 | 慢速体动 EMA（~5分钟），入睡/深睡判断 |
| `EMA_AWAKE_ALPHA` | 0.3 | 快速体动 EMA（~3秒），觉醒检测 |
| `BASELINE_BETA` | 0.01 | HR/RR 基线滚动更新速率 |

### 8.2 评分阈值

| 参数 | 值 | 用途 |
|------|----|------|
| `SLEEPINESS_THRESHOLD` | 0.45 | 入睡所需最低困倦度 |
| `HYSTERESIS_ENTER_DEEP` | 0.40 | 进入深睡所需最低评分 |
| `HYSTERESIS_EXIT_DEEP` | 0.30 | 退出深睡评分下限 |
| `HYSTERESIS_ENTER_REM` | 0.45 | 进入快速眼动所需最低评分 |
| `HYSTERESIS_EXIT_REM` | 0.30 | 退出快速眼动评分下限 |
| `CONFIDENCE_MARGIN` | 0.10 | Best Score 胜出所需的最小分差 |

### 8.3 体动阈值

| 参数 | 值 | 用途 |
|------|----|------|
| `SLEEP_EMA_MOVEMENT_MAX` | 8 | 入睡条件：慢速体动 EMA 上限 |
| `AWAKE_EMA_MOVEMENT_MIN` | 25 | 觉醒条件：快速体动 EMA 下限 |
| `FAST_AWAKE_MOVEMENT_THRESHOLD` | 45 | 快速觉醒：体动超过此值立即切换 |
| `DEEP_SLEEP_HARD_MOVEMENT_LIMIT` | 25 | 深睡/REM 体动硬上限（超过直接否决） |
| `MOVEMENT_HIGH_THRESHOLD` | 25 | 持续高体动触发觉醒 |
| `DEEP_SLEEP_MOVEMENT_THRESHOLD` | 10 | 深睡评分体动偏好上限 |
| `SLEEPINESS_MOVEMENT_THRESHOLD` | 10 | 困倦度评分体动参考上限 |
| `GETTING_UP_MOVEMENT_THRESHOLD` | 30 | 起床检测体动阈值 |

### 8.4 确认时间

| 参数 | 值（秒） | 用途 |
|------|---------|------|
| `SLEEPINESS_MIN_SEC` | 300 | 入睡需持续满足条件的秒数 |

| `LIGHT_SLEEP_CONFIRM_SEC` | 30 | 进入浅睡确认时间 |
| `AWAKE_CONFIRM_SEC` | 15 | 快速觉醒确认时间 |

| `DEEP_STABLE_MIN_SEC` | 60 | 深睡评分稳定确认时间 |
| `LIGHT_SLEEP_STABILITY_MIN_SEC` | 1200 | 进入深睡前需浅睡稳定最少时间 |

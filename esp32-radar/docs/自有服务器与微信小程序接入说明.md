# 自有服务器与微信小程序接入说明

本文对应默认 PlatformIO 环境 `freenove_esp32_s3_wroom`。

## 1. 当前固件上传链路

```text
R60ABD1 雷达 + HC-SR501 PIR
  -> ESP32-S3
  -> HTTP POST http://47.86.10.121:8080/api/radar/upload
  -> GET http://47.86.10.121:8080/api/radar/latest
  -> 微信小程序
```

默认固件不再连接 TuyaLink，也不会启动原 InfluxDB 直写任务。

## 2. ESP32 上传字段

```json
{
  "deviceId": "设备SN或MAC",
  "pirPresence": true,
  "pirValid": true,
  "radarPresence": true,
  "heartRate": 72,
  "breathRate": 16,
  "distanceCm": 145,
  "bodyMovement": 12,
  "motionStatus": 1,
  "bedStatus": "in_bed",
  "sleepState": "light_sleep",
  "sleepScore": 86,
  "dataSource": "algorithm",
  "uptimeMs": 120000
}
```

上传节奏：

- 检测到有人且生命体征有效：每 2 秒上传。
- 无人或生命体征为零：每 10 秒上传。
- PIR 状态发生变化：立即上传。
- 上传失败：每 5 秒重试。

## 3. 编译和烧录

默认自有服务器固件：

```powershell
platformio run -e freenove_esp32_s3_wroom
platformio run -e freenove_esp32_s3_wroom -t upload
platformio device monitor -b 115200
```

串口出现以下日志表示服务器接收成功：

```text
[Server] HTTP upload task started
[Server] upload succeeded, HTTP 200
```

涂鸦回退固件：

```powershell
platformio run -e tuya_freenove_esp32_s3_wroom
```

原 MQTT 平台回退固件：

```powershell
platformio run -e legacy_freenove_esp32_s3_wroom
```

## 4. 小程序页面

开发阶段在微信开发者工具中关闭请求合法域名校验，然后创建页面。

`pages/radar/radar.js`：

```javascript
const API_URL = 'http://47.86.10.121:8080/api/radar/latest'

Page({
  data: {
    radar: null,
    loading: true,
    error: ''
  },

  onLoad() {
    this.loadRadar()
  },

  onShow() {
    this.timer = setInterval(() => this.loadRadar(), 2000)
  },

  onHide() {
    clearInterval(this.timer)
  },

  onUnload() {
    clearInterval(this.timer)
  },

  loadRadar() {
    wx.request({
      url: API_URL,
      method: 'GET',
      timeout: 5000,
      success: ({ statusCode, data }) => {
        if (statusCode >= 200 && statusCode < 300) {
          this.setData({ radar: data, loading: false, error: '' })
        } else {
          this.setData({ loading: false, error: `服务器错误：${statusCode}` })
        }
      },
      fail: (error) => {
        this.setData({ loading: false, error: error.errMsg || '请求失败' })
      }
    })
  }
})
```

`pages/radar/radar.wxml`：

```xml
<view class="page">
  <view wx:if="{{loading}}">正在读取雷达数据...</view>
  <view wx:elif="{{error}}" class="error">{{error}}</view>
  <view wx:else class="card">
    <view>设备：{{radar.deviceId}}</view>
    <view>红外检测：{{radar.pirPresence ? '有人' : '无人'}}</view>
    <view>雷达检测：{{radar.radarPresence ? '有人' : '无人'}}</view>
    <view>心率：{{radar.heartRate}} BPM</view>
    <view>呼吸率：{{radar.breathRate}} 次/分钟</view>
    <view>距离：{{radar.distanceCm}} cm</view>
    <view>在床状态：{{radar.bedStatus}}</view>
    <view>睡眠状态：{{radar.sleepState}}</view>
    <view>睡眠评分：{{radar.sleepScore}}</view>
  </view>
</view>
```

`pages/radar/radar.wxss`：

```css

  padding: 32rpx;
}

.card {
  padding: 32rpx;
  line-height: 2;
  background: #fff;

}
```

在 `app.json` 的 `pages` 中加入：

```json
"pages/radar/radar"
```

## 5. 联调判断

1. 烧录默认环境并打开串口。
2. 等待 WiFi 连接和 PIR 30 秒预热。
3. 确认串口持续出现 HTTP 200。
4. 浏览器打开 `/api/radar/latest`，确认 `deviceId` 不再是模拟值 `radar001`。
5. 移动人体或改变雷达距离，确认接口字段发生变化。
6. 最后打开小程序页面，确认页面每 2 秒刷新。

正式发布小程序前，需要把服务器升级为备案域名和 HTTPS，并在微信公众平台配置 request 合法域名。

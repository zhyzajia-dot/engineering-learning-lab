# K230 二维云台视觉跟踪

## 工程文件

- `auto_pid_tracker.py`：K230 主程序，负责靶框识别、双轴 EMM 电机控制和完整 PID 自动调参。
- `main.py`：当前稳定视觉程序的保留副本。
- `host/gimbal_autotune_gui.py`：PC 图形化上位机。
- `host/start_gui.bat`：上位机启动脚本。
- `host/requirements.txt`：PC 端 Python 依赖。

## 正常使用

1. 在 CanMV IDE 中运行 `auto_pid_tracker.py`。
2. 双击 `host/start_gui.bat`，连接 COM9。
3. 上位机会自动把已保存的冠军参数加载到 K230，但不会自动启动电机。
4. 点击“开始跟踪”即可使用冠军参数；只有需要重新标定或重新寻找参数时才点击“一键自动调参”。

点击自动调参后，上位机会先通过无线链路确认 K230 的 `PONG`，确认在线后才发送 `AUTOTUNE`。单个 ESP-NOW 数据包失败会自动重试；连续 12 次无回复会明确结束请求并保存诊断日志，不会无限卡在“正在请求”。

如果更换电机、调整机械安装、交换电机方向或改变相机安装方向，应重新自动调参，不要继续使用旧方向参数。

视觉检测使用透视四边形而不是强制直角矩形：A4 从侧面看成梯形时仍可识别，并优先关联上一帧靶框，避免跳到其他轮廓。画面中的红框表示本帧真实检测，橙框表示检测短暂闪断时的缓存保持；橙框最多维持 5 帧且电机保持速度限制为 4 RPM，超时后立即停车。

当前 V10 会为 X、Y 两轴分别寻找完整的 Kp、Ki、Kd，并在 PID 冠军之后再自动寻找一组全局跟随增强参数。一次调参分为：

1. Kp、Ki、Kd 从一开始就联合粗搜，不再先固定 PD 后补积分。
2. 每组参数都执行大范围阶跃、正反向各 1.5 秒的匀速轨迹以及最终回中，统一考察大范围反应、小范围持续跟随、反向释放和静止稳定。X 轴阶跃上限为 60 px；Y 轴上限为 36 px，并会按照当前矩形四个角到画面边缘的实际余量继续自动缩小，确保完整矩形尽量留在画面中。
3. 粗搜前八名复测后，选择有参数差异的前三名进行局部搜索；半决赛再次复测，再围绕前两名进行更小步长的精细搜索。
4. 决赛保留四组有差异的参数，每组累计测试三次。排名采用中位成绩并附加最差一次惩罚，避免一次偶然好成绩成为冠军。
5. 已加载的旧冠军会在相同目标、相同测试中参加复赛；最终返回的是本轮同场胜出的两轴综合冠军。持续跟随候选的 Ki 下限为 0.08，评分会额外惩罚长时间落在运动方向后方的同号误差。
6. 在两轴 PID 冠军确定后，K230 会再测试少量“图像速度前馈 / 前馈上限 / 积分保速上限 / 大误差积分学习倍率”组合，选出更适合较快速单方向移动的跟随增强冠军。

完整过程预计约 22～32 分钟，具体取决于每次回中耗时和识别中断次数。测试范围只在画面中心附近往返，每个候选结束都会重新回中。积分在 36 px 误差区内正常学习，默认最大保速 12 RPM；大误差时降速学习而不清除已经学到的跟随速度，目标反向时积分加速释放，目标丢失时立即清零。正常跟踪还会使用有上限的图像速度前馈，缩短目标刚开始移动时的反应时间。V10 评分题目除了大幅突变和慢速连续移动，还加入了较快速单方向移动段；快速段使用连续返回轨迹，避免为了测试快速跟随而制造不真实的瞬时回中，并额外惩罚命令抖动、反复换向和回中后仍持续用力。决赛后会围绕 PID 冠军和 FOLLOW 冠军各自增加一轮少量微步长精修，争取在“快、稳、不抖”之间选出更干净的综合参数。

调参期间 X 轴最高使用 12 RPM，Y 轴最高使用 8 RPM；Y 轴回中最高使用 10 RPM。矩形任一方向接近画面边缘时会提前停车并淘汰当前候选，争取在完整丢框前自动回中；如果仍然连续丢失，则电机停止并等待重新识别，不会反复重试同一组危险参数。

V10 还会提前淘汰明显劣质候选：如果候选在缓冲时间后持续顶到轴向调参限速且误差仍然很大，或跟踪误差明显发散，就直接判负进入下一组，不再让已经输掉的参数跑完整轮。视觉跟踪会检查黑框采样点和多边覆盖，拒绝单帧中心突跳或面积明显不一致的候选；目标短暂丢失后的重新锁定按上一目标面积连续性判断，不再用画面中心硬门槛，避免 X 轴回中时把真正 A4 丢掉。

## 冠军参数

第一次成功调参会创建 `host/champion_params.json`。此后每轮仍会完整保存数据；旧冠军会被加载到 K230 并作为候选参加本轮同场测试。只有目标大小与冠军记录处在可比范围内时，才会保存本轮综合冠军：

- 使用同一个控制算法版本；
- 本轮矩形中位面积与冠军记录相近，避免拍摄距离差异造成评分失真；
- 旧冠军必须经过本轮复测，不能拿上一轮受光照、抖动影响的旧分数直接与新分数比较。

旧冠军仍可加载使用，但旧检测/评分版本与当前“透视检测 + 大幅 + 连续跟随”综合评分不可比较。第一次用当前版本完成调参时会直接建立新的完整 PID 冠军，此后才按同一评分体系守擂。

两轴分别进行冠军赛，再合并为当前综合参数。如果拍摄距离不可比，则保留历史冠军文件，但继续使用本轮参数，避免跨环境分数直接覆盖。

点击“加载冠军参数”可随时手动恢复。若确实要从零重新选冠军，关闭上位机后删除 `host/champion_params.json` 即可。

## 日志

每次自动调参都会创建 `host/logs/日期_时间/`：

- `telemetry.csv`：目标中心、误差、电机命令和单帧耗时。
- `runtime.log`：PING/AUTOTUNE 握手、K230回复、调参阶段、异常和超时原因。
- `best_params.json`：本轮 Kp/Ki/Kd、跟随增强参数、两轴综合评分、评分版本、控制版本和矩形中位面积。

日志用于诊断，不影响 K230 实时控制。K230 断电后不会自行永久保存参数，因此日常应通过上位机自动加载冠军参数。
